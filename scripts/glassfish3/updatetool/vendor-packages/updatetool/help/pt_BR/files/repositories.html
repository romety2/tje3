Repositórios e editores

A imagem de instalação é associada a um ou mais repositórios de componentes na rede. Cada repositório é representado por um editor e um URL associado. O nome do editor é único por imagem de instalação e representa a organização ou o projeto que gerencia o repositório e um escopo bem definido de conteúdo. A Update Tool mostra o editor na coluna "Origem" das exibições de listas de componentes.

Compatibilidade de software de vários repositórios

Ao criar sua própria imagem de instalação ou modificar a lista de repositórios associados a uma imagem de instalação existente, certifique-se de referenciar somente repositórios que contêm componentes compatíveis com o software existente.

Ainda que seja possível instalar suplementos e atualizações de outros repositórios, os novos suplementos ou atualizações não funcionarão adequadamente com o software existente, exceto se a organização ou projeto que gerencia os repositórios tenha validado essas combinações.

Conteúdo dos repositórios

Cada repositório inclui um catálogo de todos os componentes existentes no repositório. A Update Tool acessa catálogos mantidos localmente e catálogos baseados na rede, para listar atualizações e suplementos disponíveis.

Cada componente inclui um manifesto do respectivo conteúdo. A Update Tool acessa as informações do manifesto, para listar arquivos instalados e outras informações específicas do componente.

Editor preferencial

Um dos repositórios de editor associados à imagem de instalação deve ser identificado como editor "preferencial". Essa configuração ajuda a Update Tool, a ferramenta Software Update, o notificador da área de trabalho e a interface de linha de comando (CLI) pkg a determinar como operar quando vários repositórios de editor estão associados à imagem de instalação.

Quando um componente de suplemento existe com o mesmo nome em vários repositórios de editor e um dos editores está marcado como preferencial, a Update Tool e a CLI pkg listam e instalam o pacote com aquele nome que se encontra no repositório do editor preferencial.

Quando você usa o recurso "Mostrar versões recentes" da Update Tool ou especifica o repositório de origem durante a instalação do suplemento na CLI pkg, o suplemento é instalado no repositório de origem especificado.

Quando um componente é instalado no editor preferencial, a Update Tool, a ferramenta Software Update e o notificador da área de trabalho procuram atualizações desse componente somente no editor preferencial.

Quando o pacote é instalado em um editor não preferencial, a Update Tool, a ferramenta Software Update e o notificador da área de trabalho listam apenas as atualizações disponíveis no repositório em que o pacote foi instalado originalmente.

Modificação da lista de repositórios

Para modificar a lista de repositórios associados a uma imagem de instalação, selecione a imagem desejada e execute uma destas operações:

  • Clique com o botão direito do mouse para exibir o menu de contexto e selecione Propriedades...
  • Na guia Detalhes da imagem, selecione o botão Editar propriedades...
  • No menu principal, selecione Arquivo -> Propriedades da imagem.

Armazenamento local de informações do repositório

A lista de repositórios associados a uma imagem de instalação e cópias locais de catálogos e de manifestos de componentes são armazenadas no diretório .org.opensolaris,pkg/ de cada imagem de instalação.
