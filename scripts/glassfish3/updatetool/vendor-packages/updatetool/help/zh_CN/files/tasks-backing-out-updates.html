回退更新

“回退”是指将应用程序恢复到应用其更新之前的状态的过程。该过程包括删除作为更新的一部分安装的软件包版本，然后再安装这些软件包的所选较早版本。

执行回退的最有效方式是使用 pkg(5) 命令行界面 (Command Line Interface, CLI)。但是，执行回退前，应参考有关更新的文档，以确保您所设置的与该组件关联的所有定制内容在删除该组件时都将得以保留，且在重新安装所需的该组件的较早版本时能够被识别。大多数的组件删除和重新安装过程都能够保留用户定制的内容，但在执行回退过程之前最好先阅读相关文档。

还要考虑对整个安装映像目录进行备份。由于安装的组件和关联的许多（有时为全部）定制内容均包含于普通的安装映像目录中，所以复制安装映像目录将为您提供回退过程之前的安装状态副本。

要使用 pkg(5) 命令行界面 (Command Line Interface, CLI) 执行回退，请遵循以下步骤。

  1. 使用 pkg history -l 命令显示对应用程序映像执行的更新活动的详细日志。

  2. 往回浏览历史记录日志以查找对您要回滚的软件包执行了更改的映像更新操作。

  3. 记下映像更新操作更新的软件包。对于每个软件包，记下旧软件包版本和新软件包版本。例如，历史记录日志行

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    表明 pkg-java 软件包已从版本 1.111,0-31.2172 更新到版本 1.111,0-32.2220。

  4. 输入单个 pkg uninstall 命令，同时指定映像更新操作更新的所有软件包。例如:

    $ pkg uninstall pkg-java

  5. 输入单个 pkg install 命令，同时指定映像更新操作更新的所有软件包，包括其旧版本。例如:

    $ pkg install pkg-java@1.111,0-31.2172
