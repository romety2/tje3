系统信息库和发布者

安装映像与组件的一个或多个网络系统信息库关联。每个系统信息库均由一个发布者和一个关联的 URL 表示。对于每个安装映像，发布者名称均是唯一的，该名称既体现了管理系统信息库的组织或项目，也体现了明确的内容范围。Update Tool 在组件列表视图的“来源”列中显示发布者。

多个系统信息库中软件的兼容性

当创建自己的安装映像或修改与现有安装映像关联的系统信息库列表时，请确保只引用那些包含与现有软件兼容的组件的系统信息库。

即使您可以从其他系统信息库安装附加软件和更新，新的附加软件或更新也可能无法与您的现有软件正常配合使用，除非管理系统信息库的组织或项目已验证这些组合。

系统信息库的内容

每个系统信息库都包括一个目录，其中列出了该系统信息库中包含的所有组件。Update Tool 将访问本地维护的目录以及基于网络的目录，以列出可用的更新和附加软件。

每个组件都包括一份组件内容的清单。Update Tool 访问清单信息，以列出安装的文件及其他特定于组件的信息。

首选发布者

发布者与安装映像关联的多个系统信息库中必须有一个信息库被标识为“首选”发布者。当发布者的多个系统信息库与一个安装映像关联时，该设置可以帮助 Update Tool、Software Update 工具、桌面通知程序以及 pkg 命令行界面 (Command Line Interface, CLI) 确定如何运行。

如果某个附加软件组件以同一名称存在于发布者的多个系统信息库中，并且其中一个发布者被标记为首选发布者，则 Update Tool 和 pkg CLI 将列出首选发布者系统信息库中的该已命名软件包并进行安装。

如果使用 Update Tool 的“显示最近版本”功能或在通过 pkg CLI 安装附加软件时指定源系统信息库，则将从指定的源系统信息库安装附加软件。

从首选发布者安装组件时，Update Tool、Software Update 和桌面通知程序将仅从首选发布者查找该组件的更新。

如果是从非首选发布者安装的软件包，则 Update Tool、Software Update 和桌面通知程序将只列出最初从中安装该软件包的系统信息库中的可用更新。

修改系统信息库列表

您可以通过选择所需的映像，然后执行以下操作之一，修改与某个安装映像关联的系统信息库列表：

  • 单击鼠标右键显示上下文菜单，然后选择“属性...”。
  • 在“映像详细信息”选项卡中选择“编辑属性...”按钮。
  • 从主菜单中选择“文件”->“映像属性”。

本地存储系统信息库信息

与安装映像关联的系统信息库的列表，以及目录和组件清单的本地副本存储在每个安装映像内的 .org.opensolaris,pkg/ 目录下。
