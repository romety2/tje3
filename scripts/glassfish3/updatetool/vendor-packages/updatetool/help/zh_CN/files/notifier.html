桌面通知程序

后台桌面通知任务（或通知程序）与 Update Tool 结合使用，可以定期检查安装的 Update Tool 所知的所有安装映像有无更新，并通过桌面通知更新。

注册通知程序

如果在映像的初始安装期间选择启用自动检查和通知更新，则通知程序将自动注册为桌面环境中的启动程序。

如果初始安装程序没有注册通知程序，则只需运行 Update Tool 至少一分钟，即可自动注册通知程序。只要 Update Tool 首选项指定的更新检查频率不是“从不”，就会自动注册通知程序。

启动通知程序

每次您登录桌面时，通知程序便会启动并作为后台进程运行。只有一个或多个安装映像存在可用更新时，通知程序才会显示在桌面的通知区域中。（在 Mac OS X 上例外，在该系统上通知程序从不显示在桌面平台中）。

通知更新

当通知程序检测到存在可用更新时，它会在您的桌面通知区域显示一个图标，同时还会显示一个窗口，从该窗口中您可以启动 Software Update 工具，以便查看和应用可用的更新。

通知程序使用在您的个人 Update Tool 设置中管理的安装映像列表，来确定需要监视哪些映像和系统信息库的更新组件。通知程序检查更新的频率基于在 Update Tool 中设置的首选项。您可以通过 Update Tool 的首选项修改通知程序是否检查更新以及检查有无更新的频率：

编辑 -> 首选项

运行通知程序

根据所使用的操作系统和桌面类型的不同，将通知程序注册为桌面登录启动程序所使用的方法也有所不同：

  • Windows：“开始”菜单 ->“程序”->“启动”
  • Gnome：“系统”->“首选项”->“会话”->“启动程序”->“Update Tool 桌面通知程序”
  • Mac OS X：user name/Library/LaunchAgents/com.sun.ucnotifier.plist

当您登录桌面时，通知程序的启动程序会先查询安装映像的列表（在 Update Tool 的首选项区域中进行维护），以确定要从哪个安装映像启动通知程序任务。当您安装和卸载安装映像时，通知程序的启动程序会尝试始终使用最合适的安装映像，以从中运行通知程序。
