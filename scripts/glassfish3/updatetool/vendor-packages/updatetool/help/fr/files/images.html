Images d'installation

L'outil de mise à jour vous permet de gérer les installations d'applications de logiciels situées dans un seul répertoire du système de fichiers de votre système. Le terme d'image d'installation utilisé ci-après fait référence au répertoire général et à tous les éléments qu'il contient. Vous pouvez conserver plusieurs images d'installation sur un seul système. Aucun droit d'administration spécial n'est nécessaire pour gérer les images d'installation que vous avez installées.

Les images d'installation sont associées à un ou plusieurs référentiels de réseau pour les composants. Ces référentiels incluent des nouveaux composants et des composants mis à jour à appliquer à vos images d'installation.

Une image d'installation peut être assemblée et fournie préinstallée puis étendue à votre système ou bien elle peut être créée par une application d'installation sur votre système. De plus, vous pouvez utiliser la fonctionnalité de l'outil de mise à jour "New Image" (Nouvelle image...) pour créer votre propre image d'installation vide.

Fichier -> New Image... (Nouvelle image...)

L'Image Packaging System d'OpenSolaris fournit l'infrastructure à l'outil de mise à jour qui vous permet de gérer les images d'installation. La page de manuel de pkg(5) présente d'autres informations de référence sur Image Packaging System. Reportez-vous également au site du projet Image Packaging System pour plus d'informations :

http://opensolaris.org/os/project/pkg
