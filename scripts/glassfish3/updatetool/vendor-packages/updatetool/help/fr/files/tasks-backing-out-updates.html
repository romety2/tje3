Annulation des mises à jour

Le processus d'annulation consiste à restaurer une application dans l'état dans lequel elle était avant une mise à jour. Ce processus consiste à supprimer les versions du package installées lors de la mise à jour et à installer les versions précédentes sélectionnées de ces packages.

La façon la plus efficace de procéder à une annulation est d'utiliser l'interface de ligne de commande (CLI) de la commande pkg(5). Toutefois, avant de procéder à l'annulation, vous devriez consulter la documentation concernant la mise à jour afin de vous assurer que les éventuelles personnalisations associées seront conservées après sa suppression et qu'elles seront reconnues lors de la réinstallation de la version précédente du composant. La plupart des processus de suppression et de réinstallation conservent les personnalisations effectuées par l'utilisateur mais il est plus prudent de lire la documentation avant d'effectuer le processus d'annulation.

Regardez aussi s'il convient de faire une sauvegarde du répertoire entier de l'image d'installation. Puisque les composants installés et de nombreuses personnalisations associées, si ce n'est toutes, se trouvent dans le répertoire courant de l'image d'installation, vous disposeriez ainsi d'une copie de l'état de l'installation avant le processus d'annulation.

Pour effectuer une annulation à l'aide de l'interface de ligne de commande (CLI) de la commande pkg(5), procédez comme suit.

  1. Utilisez la commande pkg history -l pour afficher un journal détaillé des activités effectuées sur l'image de l'application.

  2. Rercherchez dans le journal d'historique l'opération image-update qui a modifié les packages que vous souhaitez restaurer.

  3. Notez les packages mis à jour par l'opération image-update. Notez l'ancienne et la nouvelle version de chaque package. Par exemple, la ligne du journal d'historique

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    indique que le package pkg-java a été mis à jour de la version 1.111,0-31.2172 à la version 1.111,0-32.2220.

  4. Saisissez la commande désinstaller le package, en précidant tous les packages mis à jour par l'opération image-update. Par exemple :

    $ désinstaller le package pkg-java

  5. Saisissez la commande pkg install, en précisant tous les packages et notamment les anciennes versions, mises à jour par l'opération image-update. Par exemple :

    $ pkg install pkg-java@1.111,0-31.2172
