Depósitos y editores

Las imágenes de instalación están asociadas a uno o varios depósitos de red de componentes. Cada depósito presenta un editor y una URL asociada. El nombre de un editor es exclusivo en cada imagen de instalación y representa tanto a la organización como al proyecto que administra el depósito y un ámbito de contenido bien definido. Update Tool muestra el editor en la columna "Origen" de las vistas de listado de componentes.

Compatibilidad de software de varios depósitos

Cuando cree su propia imagen de instalación o modifique la lista de depósitos asociados a una imagen de instalación existente, asegúrese de que consulta sólo aquellos depósitos que contienen componentes que son compatibles con el software existente.

Aunque pueda instalar complementos y actualizaciones de otros depósitos, es posible que estos no funcionen correctamente con el software existente a menos que la organización o el proyecto que administra los depósitos haya validado esas combinaciones.

Contenido de los depósitos

Cada depósito incluye un catálogo que lista los componentes que contiene. Update Tool accede a los catálogos mantenidos tanto en red como localmente para poder listar las actualizaciones y complementos disponibles.

Cada componente incluye un manifiesto de su contenido. Update Tool accede a la información de los manifiestos para listar los archivos instalados y proporcionar información específica de los componentes.

Editor preferido

Uno de los depósitos del editor asociados a una imagen de instalación debe identificarse como editor "preferido". Esta configuración ayuda a Update Tool, a la herramienta Software Update, al notificador de escritorio y a la interfaz de línea de comandos (CLI) pkg a determinar cómo será el funcionamiento cuando haya varios depósitos asociados a una imagen de instalación.

Cuando hay un componente de complemento con el mismo nombre en varios depósitos del editor y uno de estos está marcado como editor preferido, Update Tool y CLI pkg enumerarán e instalarán dicho paquete desde el depósito del editor preferido.

Cuando se utiliza la función de Update Tool "Mostrar las versiones recientes" o se especifica el depósito de origen al instalar el complemento mediante CLI pkg, el complemento se instalará desde el depósito de origen que se haya especificado.

Si se ha instalado un componente desde el editor preferido, Update Tool, Software Update y el notificador de escritorio sólo buscarán las actualizaciones de este componente desde ese editor.

Si el paquete se ha instalado desde un editor no preferido, Update Tool, Software Update y el notificador de escritorio sólo enumerarán las actualizaciones que estén disponibles desde el depósito desde el que se instaló originalmente el paquete.

Update Tool muestra de forma predeterminada sólo la versión más reciente de cada paquete. Puede utilizar el comando Ver -> Mostrar las versiones recientes para anular este comportamiento y enumerar todas las actualizaciones disponibles de todos los depósitos:

Modificación de lista de depósitos

Para modificar la lista de depósitos asociados a una imagen de instalación tiene que seleccionar la imagen en cuestión y realizar una de las siguientes operaciones:

  • Haga clic con el botón derecho para ver el menú contextual y seleccione Propiedades...
  • Seleccione el botón Editar propiedades... en la ficha Detalles de la imagen.
  • Seleccione Archivo -> Propiedades de la imagen en el menú principal.

Almacenamiento local de información sobre el depósito

La lista de depósitos que se encuentran asociados a una imagen de instalación, las copias locales de catálogos y los manifiestos se guardan en el directorio .org.opensolaris,pkg/ de cada una de las imágenes.
