To Back Out Updates

"Backing out" is the process of restoring an application to the state it was in before an update was applied. This process consists of removing the package versions installed as part of the update and then installing selected earlier versions of those packages.

The most effective way to perform a back-out is to use the pkg(5) command line interface (CLI). However, before performing the back-out, you should consult the update's documentation to ensure that any customizations you have made associated with the component will be preserved upon removal of the component and recognized during reinstallation of the desired earlier version of the component. Most component removal and reinstallation processes account for preservation of user customizations, but it is wise to read the documentation before performing the backout process.

Also consider taking a backup of the entire installation image directory. Since the installed components and many, if not all, of the associated customizations are contained within the common installation image directory, making a copy of the installation image directory will provide you a copy of the state of the installation prior to the backout process.

To perform a back-out using the pkg(5) command line interface (CLI), follow these steps.

  1. Use the pkg history -l command to display a detailed log of the update activities performed on the application image.

  2. Look backwards through the history log for the image-update operation that changed the packages you want to roll back.

  3. Note down the packages that the image-update operation updated. For each package, note down the old package version and the new package version. For example, the history log line

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    indicates that the pkg-java package was updated from version 1.111,0-31.2172 to version 1.111,0-32.2220.

  4. Enter a single pkg uninstall command, specifying all the packages that were updated by the image-update operation. For example:

    $ pkg uninstall pkg-java

  5. Enter a single pkg install command, specifying all the packages, including their old versions, that were updated by the image-update operation. For example:

    $ pkg install pkg-java@1.111,0-31.2172
