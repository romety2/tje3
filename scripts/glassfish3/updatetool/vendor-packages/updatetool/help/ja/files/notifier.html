デスクトップ通知

バックグラウンドデスクトップ通知タスク (通知) は、アップデートツールと連携動作して、インストールされているアップデートツールが認識しているすべてのインストールイメージのアップデートを定期的に確認し、アップデートがある場合はデスクトップを介して通知します。

通知の登録

イメージの初期インストール時にアップデートの自動確認および通知を有効にすると、通知はデスクトップ環境のスタートアッププログラムとして自動的に登録されます。

初期インストールプログラムで通知が登録されない場合は、アップデートツールを少なくとも 1 分間実行すると自動的に登録できます。アップデートツールの設定でアップデートの確認頻度を「確認しない」以外に指定していれば、通知は自動的に登録されます。

通知の起動

デスクトップにログインするたびに、通知が起動してバックグラウンドプロセスとして実行されます。通知は、1 つまたは複数のインストールイメージに使用可能なアップデートがある場合にのみ、デスクトップの通知領域に表示されます。(Mac OS X 上では例外であり、通知はデスクトップのドックに表示されません。)

アップデートの通知

通知によってアップデートが入手可能であることが検出されると、デスクトップの通知領域にアイコンが表示され、そのウィンドウからソフトウェアアップデートツールを起動して、入手可能なアップデートを確認して適用できます。

通知は個人のアップデートツール設定で管理するインストールイメージのリストを使用して、どのイメージおよびリポジトリのアップデートされたコンポーネントを監視するのかを決定します。通知がアップデートを確認する間隔は、アップデートツール内の設定に基づいています。通知でアップデートを確認するかどうか、およびアップデートをどれくらいの頻度で確認するかは、次の手順に従って、アップデートツールの設定で変更できます。

「編集」 -> 「設定」

通知の操作

使用しているオペレーティングシステムおよびデスクトップの種類によって、通知をデスクトップログインのスタートアッププログラムに登録する方法が異なります。

  • Windows の場合: 「スタート」メニュー -> 「プログラム」 -> 「スタートアップ」
  • Gnome の場合: 「システム設定」 -> 「設定」 -> 「セッション」 -> 「スタートアッププログラム」 -> 「アップデートツールデスクトップ通知」
  • Mac OS X: ユーザー名/Library/LaunchAgents/com.sun.ucnotifier.plist

デスクトップにログインする際、通知スタートアッププログラムはまず、アップデートツールの設定領域で維持されているインストールイメージのリストを確認して、どのインストールイメージから通知タスクを起動するのかを決定します。インストールイメージはインストールおよびアンインストールされるので、通知スタートアッププログラムは通知プログラムを実行するために、最も適したインストールイメージを常に使用するようにします。
