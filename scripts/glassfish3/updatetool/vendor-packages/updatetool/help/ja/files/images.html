インストールイメージ

アップデートツールを使用すると、システム上の単一のファイルシステムディレクトリの下に含まれているソフトウェアアプリケーションのインストールを管理できます。包括ディレクトリおよびその下に含まれるすべてのコンテンツは、インストールイメージと呼ばれます。単一のシステム上で、複数のインストールイメージを維持できます。インストールしたインストールイメージを管理するために、特別な管理用システム権限は必要ありません。

インストールイメージは、1 つ以上のコンポーネントのネットワークリポジトリと関連付けられています。これらのリポジトリには、所有しているインストールイメージに適用できる新規のおよびアップデートされたコンポーネントが含まれています。

インストールイメージは、インストール前の形式でアセンブルおよび配信してシステムに展開するか、システム上のインストールアプリケーションによって作成します。さらに、アップデートツールの「新規イメージ」機能を使用して、空のインストールイメージを作成できます。

「ファイル」 -> 「新規イメージ...」

OpenSolaris の Image Packaging System は、アップデートツールを介してインストールイメージの管理のためのインフラストラクチャーを提供します。pkg(5) のマニュアルページには、Image Packaging System に関する詳細が説明されています。また、詳細については、次の Image Packaging System プロジェクトのサイトも参照してください。
