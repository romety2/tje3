アップデートをバックアウトする

「バックアウト」は、アプリケーションをアップデートが適用される前の状態に戻す処理です。この処理は、アップデートの一部としてインストールされたパッケージバージョンを削除し、それらのパッケージの選択された前のバージョンをインストールすることから構成されます。

バックアウトをもっとも効果的に実行する方法は、pkg(5) コマンド行インタフェース (CLI) を使用する方法です。ただし、バックアウトを実行する前にアップデートのドキュメントを参照して、コンポーネントに関連して行なったカスタマイズがコンポーネントを削除しても確実に保存されるようにし、コンポーネントの前のバージョンを再インストールする際に認識されるようにしてください。ほとんどのコンポーネントの削除および再インストールの処理では、ユーザーのカスタマイズの保存について説明が示されますが、バックアウト処理を実行する前にマニュアルを読んでおくことをお勧めします。

また、インストールイメージディレクトリ全体のバックアップを行うことも検討してください。インストール済みのコンポーネントおよび、すべてではない場合でも多くの関連付けられたカスタマイズは共通インストールイメージの中に含まれており、インストールイメージディレクトリのコピーを作成するとバックアウト処理以前のインストールの状態のコピーが提供されることになります。

pkg(5) コマンド行インタフェース (CLI) を使用してバックアウトを実行するには、次の手順に従います。

  1. pkg history -l コマンドを使用して、アプリケーションイメージで実行されたアップデート処理の詳細ログを表示します。

  2. ロールバックするパッケージを変更したイメージアップデート処理の履歴ログを時間をさかのぼって確認します。

  3. イメージアップデート処理によってアップデートされたパッケージを書き留めます。パッケージごとに、古いパッケージバージョンと新しいパッケージバージョンを書き留めます。たとえば、履歴ログの

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    という行は、pkg-java パッケージがバージョン 1.111,0-31.2172 からバージョン 1.111,0-32.2220 にアップデートされたことを示します。

  4. pkg uninstall コマンドを 1 回入力し、イメージアップデート処理によってアップデートされたパッケージをすべて指定します。例:

    $ pkg uninstall pkg-java

  5. pkg install コマンドを 1 回入力し、古いバージョンも含めて、イメージアップデート 処理によってアップデートされたパッケージをすべて指定します。例:

    $ pkg install pkg-java@1.111,0-31.2172
