저장소 및 게시자

설치 이미지는 구성 요소의 하나 이상의 네트워크 저장소와 연결되어 있습니다. 각 저장소는 게시자 및 연결된 URL로 표시됩니다. 게시자 이름은 설치 이미지별로 고유하며 저장소를 관리하는 조직 또는 프로젝트와 잘 정의된 콘텐츠 범위를 모두 나타냅니다. 업데이트 도구에는 구성 요소 목록 보기의 "소스" 열에 있는 게시자가 표시됩니다.

여러 저장소의 소프트웨어 호환성

설치 이미지를 직접 생성하거나 기존 설치 이미지에 연결된 저장소 목록을 수정할 경우 기존 소프트웨어와 호환되는 구성 요소를 포함하는 저장소만 참조해야 합니다.

다른 저장소의 애드온 및 업데이트를 설치할 수 있더라도 해당 저장소를 관리하는 조직 또는 프로젝트에서 새로 설치된 애드온과 업데이트를 검증하지 않은 경우, 기존 소프트웨어에서 해당 애드온 또는 업데이트가 제대로 작동하지 않을 수 있습니다.

저장소 내용

각 저장소에는 저장소에 포함된 모든 구성 요소가 나열된 카탈로그가 포함되어 있습니다. 업데이트 도구는 사용 가능한 업데이트와 애드온의 목록을 나열하기 위해 로컬로 관리되는 카탈로그와 네트워크에 기반한 카탈로그에 액세스합니다.

각 구성 요소에는 구성 요소 콘텐츠의 매니페스트가 포함되어 있습니다. 업데이트 도구는 설치된 파일과 기타 구성 요소 고유의 정보를 나열하기 위해 매니페스트 정보에 액세스합니다.

기본 게시자

설치 이미지에 연결된 게시자의 저장소 중 하나를 "기본" 게시자로 지정해야 합니다. 이 설정을 사용하면 업데이트 도구, 소프트웨어 업데이트 도구, 데스크탑 알림 프로그램 및 pkg CLI(명령줄 인터페이스)에서 설치 이미지에 복수의 게시자 저장소가 연결되어 있는 경우에 작동하는 방법을 결정할 수 있습니다.

동일한 이름의 애드온 구성 요소가 복수의 게시자 저장소에 존재하고 해당 게시자 중 하나가 기본 게시자로 표시되는 경우 업데이트 도구 및 pkg CLI는 기본 게시자 저장소에서 명명된 패키지를 나열하고 설치합니다.

업데이트 도구의 "최신 버전 표시" 기능을 사용하거나 pkg CLI를 통해 애드온을 설치할 때 소스 저장소를 지정하면 애드온이 지정된 소스 저장소에서 설치됩니다.

기본 게시자에서 구성 요소가 설치된 경우 업데이트 도구, 소프트웨어 업데이트 및 데스크탑 알림 프로그램은 기본 게시자에서만 해당 구성 요소에 대한 업데이트를 찾습니다.

기본 게시자가 아닌 저장소에서 패키지를 설치한 경우 업데이트 도구, 소프트웨어 업데이트 및 데스크탑 알림 프로그램은 패키지를 원래 설치한 저장소에서 사용 가능한 업데이트만 나열합니다.

기본적으로 업데이트 도구는 각 패키지의 최신 버전만 나열합니다. 보기 -> 최신 버전 표시 명령을 사용하여 이 동작을 무시하고 모든 저장소에서 사용 가능한 모든 업데이트를 나열할 수 있습니다.

저장소 목록 수정

사용자는 해당 이미지를 선택하고 다음 작업 중 하나를 수행하여 설치 이미지에 연결된 저장소 목록을 수정할 수 있습니다.

  • 마우스 오른쪽 버튼을 눌러 상황에 맞는 메뉴를 표시하고 등록 정보...를 선택합니다.
  • 이미지 세부 정보 탭에서 등록 정보 편집... 버튼을 선택합니다.
  • 주 메뉴에서 파일 -> 이미지 등록 정보를 선택합니다.

저장소 정보의 로컬 스토리지

설치 이미지에 연결된 저장소의 목록뿐만 아니라, 카탈로그의 로컬 사본 및 구성 요소 매니페스트는 각 설치 이미지의 .org.opensolaris,pkg/ 디렉토리에 저장됩니다.
