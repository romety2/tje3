업데이트 되돌리기

"되돌리기"는 응용 프로그램을 업데이트가 적용되기 전의 상태로 복원하는 프로세스입니다. 이 프로세스는 업데이트의 일부로 설치된 패키지 버전을 제거한 다음 선택된 이전 버전의 패키지를 설치하는 작업으로 구성됩니다.

되돌리기를 수행하는 가장 효율적인 방법은 pkg(5) CLI(명령줄 인터페이스)를 사용하는 것입니다. 그러나 되돌리기를 수행하기 전에 업데이트의 설명서를 참조하여 구성 요소를 제거할 경우 구성 요소에 연결된 사용자 정의가 유지되는지, 또한 선택한 구성 요소의 이전 버전을 다시 설치하는 동안 사용자 정의가 인식되는지를 확인해야 합니다. 대부분의 구성 요소 설치 및 재설치 과정에서는 사용자 정의가 유지되지만, 되돌리기 프로세스를 실행하기 전에 설명서를 읽어 보는 것이 좋습니다.

또한 전체 설치 이미지 디렉토리를 백업해 두는 것도 좋습니다. 설치된 구성 요소와 여기에 연결된 대부분의(또는 모든) 사용자 정의는 같은 설치 이미지 디렉토리에 포함되어 있으므로 설치 이미지 디렉토리를 복사하면 되돌리기 프로세스 이전의 설치 상태를 사본으로 보존할 수 있습니다.

pkg(5) CLI(명령줄 인터페이스)를 사용하여 되돌리기를 수행하려면 다음 단계를 수행합니다.

  1. pkg history -l 명령을 사용하여 수행된 업데이트 작업의 세부 로그를 응용 프로그램 이미지에 표시합니다.

  2. 롤백할 패키지를 변경한 image-update 작업에 대한 내역 로그를 거꾸로 찾습니다.

  3. image-update 작업에서 업데이트한 패키지를 기록합니다. 각 패키지마다 이전 패키지 버전과 새 패키지 버전을 기록합니다. 예를 들어, 내역 로그 행

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z는

    pkg-java 패키지가 1.111,0-31.2172 버전에서 1.111,0-32.2220 버전으로 업데이트되었음을 나타냅니다.

  4. image-update 작업에서 업데이트한 모든 패키지를 지정하는 단일 pkg uninstall 명령을 입력합니다. 예:

    $ pkg uninstall pkg-java

  5. 이전 버전을 포함하여 image-update 작업에서 업데이트한 모든 패키지를 지정하는 단일 pkg install 명령을 입력합니다. 예:

    $ pkg install pkg-java@1.111,0-31.2172
