桌面通知程式

背景桌面通知作業 (或通知程式) 與 Update Tool 協同工作來定期檢查更新，當您安裝的 Update Tool 已知的所有安裝影像有更新時，會透過桌面通知您。

註冊通知程式

如果您在初始安裝影像的過程中選擇啟用自動檢查更新和通知功能，則會自動將通知程式註冊為桌面環境中的啟動程式。

如果初始安裝程式未註冊通知程式，則只需執行 Update Tool (至少需要一分鐘的時間)，即可自動註冊通知程式。只要您的 Update Tool 喜好設定指定的更新檢查頻率不是 [永遠不]，便會自動註冊通知程式。

啟動通知程式

每次登入桌面時，通知程式均會啟動並作為背景程序執行。只有當您的一個或多個安裝影像有可用的更新時，通知程式才會顯示在桌面的通知區域中。(Mac OS X 例外，通知程式永遠不會顯示在 Mac OS X 桌面的停駐區域中)。

更新通知

當通知程式偵測到有可用的更新時，會在桌面的通知區域顯示一個圖示並會顯示一個視窗，您可以從該視窗啟動 Software Update 工具，以查看和套用可用的更新。

通知程式會使用在個人 Update Tool 設定中管理的安裝影像的清單，來決定要為更新元件監視哪些影像和儲存庫。通知程式檢查是否有更新的頻率取決於 Update Tool 內設定的喜好設定。您可以透過 Update Tool 的喜好設定來修改通知程式是否檢查更新以及檢查更新的頻率：

[編輯] -> [喜好設定]

通知程式作業

根據所使用的作業系統與桌面類型不同，將通知程式註冊為桌面登入啟動程式所使用的方法也不同：

  • Window：[開始] 功能表 -> [程式集] -> [啟動]
  • Gnome：[系統] -> [喜好設定] -> [階段作業] -> [啟動程式] -> [Update Tool 桌面通知程式]
  • Mac OS X：user name/Library/LaunchAgents/com.sun.ucnotifier.plist

登入桌面後，通知程式啟動程式會先查看 Update Tool 喜好設定區域中維護的安裝影像清單，以確定從中啟動通知程式作業的安裝影像。安裝和解除安裝安裝影像時，通知程式啟動程式會嘗試自動使用最適當的安裝影像，以從中執行通知程式。
