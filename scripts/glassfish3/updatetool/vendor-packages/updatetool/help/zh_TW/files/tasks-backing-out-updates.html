復原更新

「取消作業」是將應用程式復原到套用更新之前的狀態的過程。此過程包括移除作為更新的一部分安裝的套裝軟體版本，然後安裝這些套裝軟體的已選取舊版。

執行取消作業最有效的方式是使用 pkg(5) 指令行介面 (CLI)。但是，在執行取消作業之前，應該查閱有關更新的文件，以確保在移除元件時保留對該元件所做的任何自訂，並確保在重新安裝所需的舊版元件時可以辨識出這些自訂。大多數元件的移除與重新安裝程序會保留使用者的自訂，但還是有必要在執行復原程序前先閱讀文件。

也應當考慮備份整個安裝影像目錄。由於共用安裝影像目錄中包含已安裝元件和許多相關的自訂 (有時是全部自訂)，因此複製安裝影像目錄後，便能在復原程序前先提供安裝狀態的副本。

若要使用 pkg(5) 指令行介面 (CLI) 執行取消作業，請遵循以下步驟：

  1. 使用 pkg history -l 指令顯示對應用程式影像執行的更新作業的詳細記錄。

  2. 往回瀏覽歷程記錄，以查找對您要復原的套裝軟體進行了變更的影像更新作業。

  3. 記下該影像更新作業所更新的套裝軟體。對於每個套裝軟體，請將舊套裝軟體版本和新套裝軟體版本均記下來。例如，歷程記錄行

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    指示 pkg-java 套裝軟體已從版本 1.111,0-31.2172 更新到了版本 1.111,0-32.2220。

  4. 輸入單個 pkg uninstall 指令，指定已由該影像更新作業更新的所有套裝軟體。例如：

    $ pkg uninstall pkg-java

  5. 輸入單個 pkg install 指令，指定已由該影像更新作業更新的所有套裝軟體 (包括其舊版)。例如：

    $ pkg install pkg-java@1.111,0-31.2172
