儲存庫和發佈者

安裝影像與元件的一或多個網路儲存庫相關。每個儲存庫均以一個發佈者和一個關聯的 URL 表示。每個安裝影像的發佈者名稱都是唯一的，它表示管理該儲存庫的組織或專案以及明確定義的內容範圍。Update Tool 在元件清單檢視的 [來源] 欄中顯示發佈者。

多個儲存庫中軟體的相容性

當您建立自己的安裝影像或修改與現有安裝影像相關聯的儲存庫清單時，請確保僅參照那些包含與現有軟體相容之元件的儲存庫。

即使您可能可以安裝來自其他儲存庫的附加元件和更新，但除非管理儲存庫的組織或專案已驗證過這些組合，否則新的附加元件或更新可能無法與現有軟體正常配合使用。

儲存庫目錄

每個儲存庫均包括一個目錄，其中列出了儲存庫中包含的所有元件。Update Tool 可同時存取本機維護的目錄與以網路為基礎的目錄，以列出可用的更新與附加元件。

每個元件均包括元件內容的清單。Update Tool 會存取清單資訊，以列出安裝的檔案與其他元件特定的資訊。

喜好的發佈者

其中一個與安裝影像相關聯的發佈者儲存庫必須標識為「喜好的」發佈者。此設定可以幫助 Update Tool、Software Update 工具、桌面通知程式和 pkg 指令行介面 (CLI) 確定當多個發佈者儲存庫與一個安裝影像相關聯時如何作業。

如果名稱相同的附加元件出現在多個發佈者儲存庫中且其中一個發佈者被標記為喜好的發佈者，Update Tool 和 pkg CLI 會列出和安裝喜好的發佈者儲存庫中指定的套裝軟體。

如果使用 Update Tool 的 [顯示最近的版本] 功能或在透過 pkg CLI 安裝附加元件時指定來源儲存庫，則會從您指定的來源儲存庫安裝附加元件。

如果元件是從喜好的發佈者安裝的，則 Update Tool、Software Update 和桌面通知程式將僅從喜好的發佈者中查找該元件的更新。

如果套裝軟體是從非喜好的發佈者安裝的，則 Update Tool、Software Update 和桌面通知程式僅會列出最初從其中安裝套裝軟體的儲存庫中的可用更新。

依預設，Update Tool 僅會列出每個套裝軟體的最新版本。您可以使用 [檢視] -> [顯示最近的版本] 指令來置換此運作方式，並列出所有儲存庫中的所有可用更新。

修改儲存庫清單

您可以修改與安裝影像相關聯的儲存庫清單，方法是選取所需的影像，然後執行以下作業之一：

  • 按一下滑鼠右鍵以顯示右鍵功能表，並選取 [特性...]
  • 在 [影像詳細資訊] 標籤上選取 [編輯特性...] 按鈕。
  • 從主功能表中選取 [檔案] -> [影像特性]。

儲存庫的本機儲存資訊

與安裝影像相關聯的儲存庫的清單，以及目錄與元件清單的本機複本，均位在每個安裝影像內的 .org.opensolaris,pkg/ 目錄下。
