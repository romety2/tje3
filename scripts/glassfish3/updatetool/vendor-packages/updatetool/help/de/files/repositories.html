Repositorys und Publisher

Installationsabbilder sind einem oder mehreren Netzwerk-Repositories mit Komponenten zugeordnet. Jedes Repository wird mit einem Publisher und einer verknüpften URL dargestellt. Ein Publisher-Name ist in jedem Installationsabbild eindeutig und steht sowohl für das Unternehmen als auch für das Projekt, über das das Repository und ein genau definierter Bereich des Inhalts verwaltet werden. Das Update Tool zeigt den Publisher in der Spalte "Quelle" der Ansichten für die Komponentenauflistung an.

Kompatibilität von Software aus mehreren Repositories

Wenn Sie ein eigenes Installationsabbild erstellen oder die Liste der Repositories ändern, die mit einem bestehenden Installationsabbild verknüpft sind, müssen Sie darauf achten, nur auf die Repositories zu verweisen, deren Komponenten mit der bestehenden Software kompatibel sind.

Auch wenn Sie Add-Ons und Update aus anderen Repositories installieren können, funktionieren die neuen Add-Ons oder Updates unter Umständen nicht einwandfrei mit der bestehenden Software, wenn das Unternehmen oder das Projekt, das die Repositories verwaltet, diese Kombinationen nicht validiert hat.

Inhalt der Repositories

Jedes Repository beinhaltet einen Katalog, in dem alle im Repository enthaltenen Komponenten aufgeführt sind. Update Tool greift sowohl auf lokal verwaltete als auch auf netzwerkbasierte Kataloge zu, um die verfügbaren Updates und Add-Ons aufzulisten.

Jede Komponente beinhaltet ein Manifest des Komponenteninhalts. Update Tool greift auf die Manifestinformationen zu, um installierte Dateien und andere komponentenspezifische Informationen aufzulisten.

Bevorzugte Publisher

Eines der mit einem Installationsabbild verknüpften Repositorys muss als "bevorzugtes" Repository gekennzeichnet werden. Mit dieser Einstellung können Update Tool, das Software Update Tool, der Desktop-Meldungsgenerator und die pkg-Befehlszeilenschnittstelle ermitteln, wie sie vorgehen müssen, wenn mit einem Installationsabbild mehrere Repositorys verknüpft sind.

Wenn eine Add-On-Komponente dem Namen nach in mehreren Publisher-Repositorys vorliegt und einer der Publisher als bevorzugter Publisher gekennzeichnet ist, führen Update Tool und die pkg-Befehlszeilenschnittstelle das bezeichnete Paket auf und installieren es aus dem bevorzugten Publisher-Repository.

Wenn Sie die Funktion "Alle Versionen anzeigen" von Update Tool verwenden oder Sie beim Installieren des Add-Ons über die pkg-Befehlszeilenschnittstelle das Quell-Repository angeben, wird das Add-On über das angegebene Quell-Repository installiert.

Wenn eine Komponente über den bevorzugten Publisher installiert wurde, suchen Update Tool, Software Update und der Desktop-Meldungsgenerator nach Updates für diese Komponente, die nur vom bevorzugten Publisher stammen.

Wenn das Paket aus einem nicht bevorzugten Repository installiert wurde, führen Update Tool, Software Update und der Desktop-Meldungsgenerator nur die Updates auf, die im Repository verfügbar sind, über das das Paket ursprünglich installiert wurde.

Update Tool listet standardmäßig nur die neueste Version jedes Pakets auf. Mit dem Befehl Anzeigen -> Alle Versionen anzeigen können Sie dieses Verhalten außer Kraft setzen und alle verfügbaren Updates aus allen Repositorys auflisten:

Ändern der Liste der Repositories

Sie können die Liste der Repositories, die mit einem Installationsabbild verknüpft sind, bearbeiten, indem Sie das gewünschte Abbild auswählen und einen der folgenden Vorgänge ausführen:

  • Klicken Sie mit der rechten Maustaste, um das Kontextmenü anzuzeigen, und wählen Sie Eigenschaften... aus.
  • Klicken Sie auf der Registerkarte Abbilddetails auf die Schaltfläche Abbilddetails.
  • Wählen Sie im Hauptmenü Datei -> Abbildeigenschaften aus.

Locale Speicherung der Repository-Informationen

Die Liste der einem Installationsabbild zugeordneten Repositories sowie lokale Kopien von Katalogen und Komponentenmanifesten befinden sich im Verzeichnis .org.opensolaris,pkg/ der einzelnen Installations-Abbilder.
