So heben Sie Updates auf

"Backing Out" ist der Prozess des Wiederherstellens einer Anwendung, sodass sie wieder in den Vor-Update-Status zurückkehrt. Dieser Prozess besteht aus dem Entfernen der Paketversionen, die als Bestandteil des Updates installiert wurden, und dem anschließenden Installieren der zuvor ausgewählten Versionen dieser Pakete.

Die optimale Möglichkeit zum Ausführen eines Back-Outs ist die Verwendung der pkg(5)-Befehlszeilenoberfläche. Vor dem Ausführen eines Back-Outs sollten Sie die zu der Komponente gehörende Dokumentation zurate ziehen, um sicherzustellen, dass alle Anpassungen, die Sie an der Komponente vorgenommen haben, beim Entfernen der Komponente erhalten bleiben und während der erneuten Installation der gewünschten früheren Version der Komponente erkannt werden. Bei den meisten Prozessen zum Entfernen und erneuten Installieren wird die Beibehaltung von Benutzeranpassungen gewährleistet, es ist jedoch anzuraten, vor Durchführung des Backout-Vorgangs die Dokumentation zu lesen.

Außerdem sollte eine Sicherungskopie des gesamten Installationsabbildverzeichnisses erstellt werden. Da die installierten Komponenten und zahlreiche, wenn nicht alle zugehörigen Anpassungen sich innerhalb des gemeinsamen Installationsabbildverzeichnisses befinden, erhalten Sie durch eine Kopie des Installationsabbildverzeichnisses eine Kopie des Installationsstatus vor dem Backout-Prozess.

Führen Sie die folgenden Schritte aus, um mithilfe der pkg(5)-Befehlszeilenoberfläche ein Back-Out auszuführen.

  1. Verwenden Sie den pkg history -l-Befehl, um ein detailliertes Protokoll der Update-Aktivitäten anzuzeigen, die am Anwendungsabbild ausgeführt werden.

  2. Gehen Sie das Verlaufsprotokoll rückwärts bis zumimage-update-Vorgang durch, mit dem die Pakete geändert wurden, an denen ein Roll-Back ausgeführt werden soll.

  3. Notieren Sie sich die Pakete, die den Vorgang image-update aktualisiert haben. Notieren Sie sich für jedes Paket die alte Paketversion und die neue Paketversion. Diese Zeile im Verlaufsprotokoll

    pkg:/pkg-java@1.111,0-31.2172:20090608T230741Z -> pkg:/pkg-java@1.111,0-32.2220:20090703T010710Z

    gibt beispielsweise an, dass das Paket pkg-java von Version 1.111,0-31.2172 auf Version 1.111,0-32.2220 aktualisiert wurde.

  4. Geben Sie einen einzelnen pkg install-Befehl ein, mit dem Sie alle Pakete angeben, die vom image-update-Vorgang aktualisiert wurden. Beispiel:

    $ pkg uninstall pkg-java

  5. Geben Sie einen einzelnen pkg install-Befehl ein, mit dem Sie alle Pakete, einschließlich ihrer alten Versionen angeben, die vom Vorgang image-update aktualisiert wurden. Beispiel:

    $ pkg install pkg-java@1.111,0-31.2172
